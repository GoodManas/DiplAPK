# DP — мобильное приложение исполнителя

Expo-приложение для выездных специалистов. Работает вместе с локальным API-сервером (порт **3001**). Без сервера вход и заявки недоступны.

## Что нужно на новом ПК

| Компонент | Зачем |
|-----------|--------|
| [Node.js](https://nodejs.org/) LTS (20+) | `npm`, сервер, Expo |
| Git (или архив проекта) | скопировать код |
| Телефон + [Expo Go](https://expo.dev/go) | запуск приложения |
| Одна Wi‑Fi сеть | ПК и телефон должны видеть друг друга |

На Windows для модуля `better-sqlite3` иногда нужны **Visual Studio Build Tools** (C++). Если `npm install` в `server/` падает с ошибкой компиляции — установите [Build Tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/) с workload «Desktop development with C++» и повторите установку.

## 1. Перенос проекта

```bash
git clone <URL-репозитория> DP
cd DP
```

Либо скопируйте папку целиком (флешка, облако). Папку `node_modules` копировать не обязательно — зависимости ставятся заново.

## 2. Установка зависимостей

В корне проекта:

```bash
npm install
cd server
npm install
cd ..
```

Опционально — веб-панель диспетчера:

```bash
cd dispatcher
npm install
cd ..
```

## 3. Настройка адреса API

Приложение на телефоне **не** ходит на `localhost` вашего ПК — нужен IP в локальной сети.

1. Узнайте IPv4 ПК: `ipconfig` (Windows) → строка **IPv4**, например `192.168.1.105`.
2. Создайте файл `.env` в корне (можно скопировать `.env.example`):

```env
EXPO_PUBLIC_API_URL=http://192.168.1.105:3001
```

Подставьте **свой** IP. После смены `.env` перезапустите `npm start`.

Разрешите входящие подключения к порту **3001** в брандмауэре Windows, если телефон не достучится до API.

## 4. База и демо-данные (первый запуск)

```bash
npm run server:seed
```

Создаётся SQLite в `server/data/`. Тестовые учётки:

| Роль | Код сотрудника | Пароль |
|------|----------------|--------|
| Исполнитель (мобильное приложение) | `405IS` | `demo123` |
| Диспетчер (веб) | `DISPATCHER` | `demo123` |

Повторный `seed` не затирает уже созданных пользователей (INSERT OR IGNORE).

## 5. Запуск

**Терминал 1 — API:**

```bash
npm run server
```

Проверка: в браузере на ПК откройте `http://localhost:3001/api/health` — ответ `{"ok":true,...}`.

**Терминал 2 — мобильное приложение:**

```bash
npm start
```

В консоли появится QR-код.

**На телефоне:**

1. Тот же Wi‑Fi, что и у ПК.
2. Откройте **Expo Go** → сканируйте QR (Android) или камера / Expo Go (iOS).
3. Войдите: **405IS** / **demo123**.

Если «сеть недоступна» или ошибки API — проверьте IP в `.env`, брандмауэр и что сервер запущен.

## 6. Опционально: диспетчер

```bash
npm run dispatcher
```

Панель: `http://localhost:5173`. Логин: **DISPATCHER** / **demo123**.

Чтобы диспетчер ходил на тот же API, в `dispatcher` можно задать `VITE_API_URL=http://localhost:3001` (файл `.env` в папке `dispatcher`).

## Полезные команды

| Команда | Описание |
|---------|----------|
| `npm start` | Expo (QR для телефона) |
| `npm run android` | эмулятор Android (нужен Android Studio) |
| `npm run server` | API + WebSocket |
| `npm run server:seed` | демо-БД и заявки |
| `npm run dispatcher` | веб диспетчера |

## Структура (кратко)

- `src/` — экраны мобильного приложения (Expo)
- `server/` — Node.js API, SQLite
- `dispatcher/` — веб для диспетчера

Подробнее об архитектуре: [docs/architecture.md](docs/architecture.md).

## Частые проблемы

- **Приложение не логинится** — неверный IP в `.env`, сервер не запущен, телефон не в той же сети.
- **После смены `.env` старый URL** — остановите Expo (`Ctrl+C`) и снова `npm start`.
- **Порт 3001 занят** — закройте другой процесс или задайте `PORT=3002` при запуске сервера и обновите URL в `.env`.
