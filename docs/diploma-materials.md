## Реализованный функционал (для записки)

**Авторизация и роли:** JWT, роли `dispatcher` / `executor`, матрица прав (`server/src/roles.ts`), смена пароля, сохранение сессии на телефоне (AsyncStorage).

**Диспетчер:** сводка (статистика), заявки с поиском и фильтрами, приоритет (низкий/обычный/срочный), телефон клиента, назначение и отмена, история по заявке, управление сотрудниками.

**Исполнитель (мобильное):** фильтры и поиск, приоритет, звонок клиенту, маршрут, фото результата, история статусов, профиль и смена пароля.

## Тест-кейсы

| № | Действие | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Вход DISPATCHER / demo123 | Открывается панель диспетчера |
| 2 | Создание заявки с приоритетом «Срочный» | Заявка в статусе «Новая» |
| 3 | Назначение на 405IS | Статус «Назначена» |
| 4 | Вход 405IS в Expo Go | Видна назначенная заявка |
| 5 | «Начать работу» | Статус «В работе», история в БД |
| 6 | Фото + «Завершить» | Статус «Выполнена», запись с фото в истории |
| 7 | Вкладка «Сводка» у диспетчера | Отображаются счётчики по статусам |
| 8 | Добавление сотрудника | Новый исполнитель в списке |
| 9 | Смена пароля в профиле | Успешный вход с новым паролем |
| 10 | WebSocket + polling | Диспетчер видит смену статуса без перезагрузки |

## Экономическая эффективность (шаблон)

Укажи в записке:

- сокращение времени передачи статуса диспетчеру (было: звонок, стало: синхронизация в системе);
- снижение ошибок из-за бумажных журналов;
- ориентировочная стоимость разработки (часы × ставка) vs. экономия ФОТ/простоя техники.

## Запуск для демонстрации на защите

Три терминала в папке `C:\Users\PC\vscode\DP`:

```powershell
npm run server:seed
npm run server
npm run dispatcher
```

Четвёртый — мобильное приложение:

```powershell
npm start
```

На телефоне в `.env`: `EXPO_PUBLIC_API_URL=http://ВАШ_IP:3001`

Диспетчер: http://localhost:5173
